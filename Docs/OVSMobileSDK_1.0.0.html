OVS Mobile SDK Manual - Version 1.0.0

Note: All API use .ovs file as data source

Overview

OVS Mobile SDK Manual is used to show developer how to use OVSKit library for your project to facilitate the process of developing a project between server and mobile application. As we know that, most of the time, when we start to create a mobile application. We are more likely to set up a server to keep the records of the user in the server for later push notification, login or other logic process. Those kind of logic requires a lot of efforts in the server side and the mobile side. Now, OVSKit comes out. It helps you to short the develop process of your app. Without setting up a server, you can simply integrate the OVSKit library to your project and just did some simply configuration, you are going to have the ability to manage your mobile app's user, push notification, commerial goods management and so on.

We offer OpenVS Platform to manage your mobile app under a double encryption, independent private key/pair data transmission and store secure enviornment. Therefore, you don't need to worry about your data secure issue. Moreover, we open our server side source code (which is written by java) and mobile app source code (iOS and Android) for the people who want to improve the feature and shorten the development process. Even more exciting, online payment solution are provided in the OVSKit. You can easily do the online purchase feature by using our OVSKit without adding other extra library or security checking. All the available open source code (server code and mobile app ) are forever free and we will maintain it forever as well.

If you want to download the source code of OVSKit, please create an account here. Then, login to your account and create an application in our web dashboard. After you successfully create an application by using our dashboard. You can download all the source code on the side of the application bar.

Get Started

Step 1: Initialize SDK

When you start to use this SDK. The first thing is to set up the Application Key for your app in your mobile app project. You can get the Application Key from OpenVS Dashboard by open https://dashboard.openvs.org. This app key has to be set up first in any situation. When set up the app key, the callback function will nofity you it is a valid app key or not. You don't need to wait for the callback function to run your app if you know the app key is always a valid app key. If the app key is not valid, your following OVSKit api call will be failed.

Note: Even you download the source code (server code, and mobile app code), you still need to set up a valid key. because OVSKit will talk to OpenVS server every time when setAppKey function get called. After that, all the OVSKit api call will point to your server.

//set up your app key in the AppDelegate.swift file func application(_ application: UIApplication, didFinishLaunchingWithOptions launchOptions: [UIApplicationLaunchOptionsKey: Any]?) -> Bool { OVSKit.setAppKey("<your-ovs-app-key>") return true }
Not Available Now

Step 2: Assign Server URL

This is an optional step. It is used to define the server url to receive all the API calls. If you download our server code and deploy the server code in your own server. You need to set up the serverURL variable. Otherwise, there is no need to define your own server url. The default server url will be pointed to the https://api.openvs.org. If you have our openvs platform service, just keep default server url. All the API call should works properly. If you meet issues, please go to our discussion section of openvs.org via https://www.openvs.org/discussion

//set up your app key in the AppDelegate.swift file func application(_ application: UIApplication, didFinishLaunchingWithOptions launchOptions: [UIApplicationLaunchOptionsKey: Any]?) -> Bool { OVSKit.setAppKey("<your-ovs-app-key>") OVSKit.setServerURL("http://mylocalserver.com") return true }
Not Available Now

Step 3: Register Device to Server

After you finish Step 1 and Step 2. Now, it is the time for you to register your device information to your server. so that. your server can update your device push notification and device id for the furture push notification and other statistic feature. Otherwise, some functions may not work properly in openvs app. We recommend you to put the code in the app active callback function. Because after you successfully register you device to the server. Server will return a configuration for your app based on your Application Settings. This configuration will be loaded the next time the app turns to be active. For you, you don't need to worry about the configuration. It will be handled by the OVSKit.

//set up your app key in the AppDelegate.swift file func applicationDidBecomeActive(_ application: UIApplication) { OVSKit.registerDevice() }
Not Available Now

Step 4: Use Push Notification

If you are going to use the push notification in the app by using the OVSKit. you can use the build in function from OVSKit to register the push notification in both Apple And Google notification server. After that, listen to the callback function from Apple and Google server to prepare the device token for OVSKit. Details are as below:

// need to call registerPushNotification in the place where you want to pop up // push notification permission dialog to register the push notification. // here we put it in the didFinishLaunching delegate function func application(_ application: UIApplication, didFinishLaunchingWithOptions launchOptions: [UIApplicationLaunchOptionsKey: Any]?) -> Bool { OVSKit.setAppKey("THIS-IS-A-DEV-APP-KEY") OVSKit.registerPushNotification() return true } // implement the delegate in AppDelegate.swift to prepare device token. once a valid // push nontification device token comes back. func application(_ application: UIApplication, didRegisterForRemoteNotificationsWithDeviceToken deviceToken: Data) { OVSKit.preparePushNotification(deviceToken) }

appKey: string | AppKey generated from OVS Dashboard callback: (boolean isValid) | callback function from ovs server, indicate the appKey is valid or not

//set up your app key in the AppDelegate.swift file func application(_ application: UIApplication, didFinishLaunchingWithOptions launchOptions: [UIApplicationLaunchOptionsKey: Any]?) -> Bool { OVSKit.setAppKey("<your-ovs-app-key>") { (isValid) in //do something here to check the isValid flag } return true }
Not Available Now

setServerURL(serverURL)

serverURL: string | the API server url, where you want to point to. use the format: https://your.domain.com or https://your.ip.address return: void

This function helps you set up your server url. If you decide to use openvs platform, you don't need to change the server url. If you use our open source server code and set up your own server, you need to set up this server url to your server ip address or domain in AppDelegate.swift file in iOS.

//set up your app key in the AppDelegate.swift file func application(_ application: UIApplication, didFinishLaunchingWithOptions launchOptions: [UIApplicationLaunchOptionsKey: Any]?) -> Bool { OVSKit.setAppKey("<your-ovs-app-key>") OVSKit.setServerURL("http://your.ip.address") return true }
Not Available Now

registerDevice()

return: void

This function is used to initialize the app based on the configuration from server. We highly recommend you to put this code in the didBecomeActive function of the AppDelegate file in iOS platform. So that everytime, when the app is active, you can get the update of the configuration. The configuration includes the flow control of the pages in the app, splash advertisements control and so on. If the app frequently go to background and return back to foreground. This function smartly decides the waiting duration in terms of sending the request to remote server. The duration for now is one minute. You can change it remotely in the server configuration.

//set up your app key in the AppDelegate.swift file func applicationDidBecomeActive(_ application: UIApplication) { OVSKit.registerDevice() }
Not Available Now

registerPushNotification()

return: void

if you want your app to receive push notification from server. You need to call this function when the app starts. There is no fixed place to put this function. But once you call this function, it will pop up a push notification dialog to require user's permission when the first time the user installs the app. Once you call this function, you are going to receive server remote push notification in the future. Moreover, this function can not stay alone. Whenever you call this function, you need to call preparePushNotification(deviceToken:data) function in AppDelegate as well. so that, the results of registering the push notification can comes back and we can get the push notification device token. After the first time you call this function, in the showDeviceInfo() function, you can see the push notification is not empty anymore.

func application(_ application: UIApplication, didFinishLaunchingWithOptions launchOptions: [UIApplicationLaunchOptionsKey: Any]?) -> Bool { OVSKit.setAppKey("THIS-IS-A-DEV-APP-KEY") OVSKit.registerPushNotification() return true }
Not Available Now

preparePushNotification(deviceToken:data)

deviceToken: data | It is the binary format device token. Which is from didRegisterForRemoteNotificationsWithDeviceToken callback function return: void

This function brings the received device token and send to server for storing. It has be used in pair of the function * registerPushNotification*. Once this function gets called. you should find the registered device token in the web dashboard.

func application(_ application: UIApplication, didRegisterForRemoteNotificationsWithDeviceToken deviceToken: Data) { OVSKit.preparePushNotification(deviceToken) }

Questions & Answers

Frequently Asks

◘ Where can I get the source code?

After you register an account in OpenVS.org, then you can find the source code (server, ios, android) in the application information page. If you didn't create an application in OpenVS.org. you can not find the source code. Moreover, each application has an AppKey after you successfully create one. please use that AppKey in your project. Otherwise, your project won't work. We make our project open source and forever free. But we need you to set App Key so that we can guarantee we have monitor of our distributed code to avoid some people amuse our source code.

◘ Do I need to pay for OVS service, what is the price?

OpenVS (OVS) is open source and forever free. We won't charge you any fee if you set up your own server and manage the server and source code yourself. However, if you decide to use our OpenVS Management Platform (OpenVS Platform) to control and maintain your app. We have several package for you. The basic package is free for everyone, you can find the detailed information in this link: https://www.openvs.org/price

◘ Where can I get the technical support in terms of using OVS?

We have support email and forum for you to find your answer and questions. Everytime, you can search our forum for your issue or problems. If you can not find the answer, you can send a support email to us, We will reply you within 24 hours. Our forum is in this link: https://www.openvs.org/forum

◘ Can I modify the source code of OVS and release to my client?

Yes, feel free to modify the source code and release the product which build by modified version of OVS to your client. We won't charge you anything from your final product and we won't charge you the fee for your commerial use. However, you can not sells or distribute your modified version of the source code to others, except us. That is to say, you can modify the source code for building up your product. You CAN NOT modify the source code and distribute the modified version as a commerial product.

◘ Do I need to open source after I modify OVS source code?

No, you don't need to open source your modified version of OVS source code. And you are not permitted by us to open source your modified version as well. We have the right to let you share the source code with us if we want to check your modified version. However, you always own the copyright of your modified part of your OVS source code.

◘ What is the difference between the paid version and free version

There is no functionalities difference between the paid version and free version. The only differences are that:
[1] paid version supports data encryption between server and mobile app
[2] paid version supports data/user personal information encryption in database
[3] paid version provides 7/24 running server to you
[4] paid version allows you have online payment service
[5] paid version allows you manage more than one apps in the web dashboard
